# 基于LSTM的献血预测模型产品文档

## 1. 项目概述

### 1.1 项目背景
无偿献血是医疗系统的重要组成部分，确保血液供应的稳定性对于医疗机构和患者至关重要。然而，献血者的献血行为具有一定的不确定性，如何预测献血者的献血行为，特别是高频率献血者的献血意愿，是献血机构面临的重要挑战。

### 1.2 项目目标
开发一个基于LSTM（长短期记忆网络）的献血预测模型，利用历史献血数据预测献血者是否会在未来特定时间点献血，帮助献血机构更有效地规划献血活动，提高血液采集效率。

### 1.3 技术方案
借鉴彩票预测系统中的LSTM实现方式，结合献血数据的特点，构建一个专门针对献血预测的深度学习模型。

## 3. 模型设计

### 3.1 模型架构
基于LSTM的献血预测模型架构如下：

```
输入层 → 嵌入层 → 双向LSTM层 → 多头注意力层 → 全连接层 → 输出层
```

- **输入层**：接收标准化后的输入特征
- **嵌入层**：将输入特征转换为密集向量表示
- **双向LSTM层**：捕获序列数据的前后依赖关系
- **多头注意力层**：增强模型对重要特征的关注
- **全连接层**：进行特征融合和转换
- **输出层**：使用sigmoid激活函数，输出二分类预测结果

### 3.2 模型参数
| 参数 | 描述 | 建议值 |
|------|------|--------|
| 输入特征维度 | 输入特征的数量 | 4 |
| 嵌入维度 | 嵌入层的输出维度 | 50 |
| LSTM隐藏层大小 | LSTM层的隐藏状态维度 | 256 |
| LSTM层数 | LSTM层的数量 | 2 |
| 注意力头数 | 多头注意力的头数 | 8 |
|  dropout率 | 正则化参数 | 0.1 |
| 学习率 | 优化器的学习率 | 0.001 |
| 批量大小 | 训练批次大小 | 32 |
| 训练轮数 | 模型训练的轮数 | 100 |

### 3.3 特征工程
1. **特征标准化**：对输入特征进行标准化处理，确保模型训练的稳定性
2. **特征衍生**：
   - 献血频率：总献血次数 / 首次献血至今的月数
   - 献血密度：总献血量 / 首次献血至今的月数
   - 献血间隔：上次献血至今的月数
3. **序列构建**：将献血者的历史数据构建为时序序列，用于LSTM模型输入

### 4.2 模型实现
```python
import torch
import torch.nn as nn
import torch.optim as optim
import numpy as np
import pandas as pd
from sklearn.preprocessing import StandardScaler
from sklearn.model_selection import train_test_split

class BloodDonationLSTM(nn.Module):
    def __init__(self, input_size, output_size=1, hidden_size=256, num_layers=2, num_heads=8, dropout=0.1, embedding_dim=50):
        super(BloodDonationLSTM, self).__init__()
        # 嵌入层
        self.embedding = nn.Embedding(100, embedding_dim)  # 用于处理可能的分类特征
        # 双向LSTM
        self.lstm = nn.LSTM(input_size, hidden_size, num_layers, dropout=dropout, batch_first=True, bidirectional=True)
        self.dropout = nn.Dropout(dropout)
        # 多头注意力机制
        self.MultiheadAttention = nn.MultiheadAttention(embed_dim=hidden_size*2, num_heads=num_heads, dropout=dropout)
        # 输出层
        self.linear = nn.Linear(hidden_size*2, output_size)
        self.sigmoid = nn.Sigmoid()  # 用于二分类

    def forward(self, x):
        # x: [batch_size, seq_len, input_size]
        lstm_out, _ = self.lstm(x)  # [batch_size, seq_len, hidden_size*2]
        lstm_out = self.dropout(lstm_out)
        
        # 应用多头注意力
        lstm_out = lstm_out.permute(1, 0, 2)  # [seq_len, batch_size, hidden_size*2]
        context_vector, _ = self.MultiheadAttention(lstm_out, lstm_out, lstm_out)
        context_vector = context_vector.permute(1, 0, 2)  # [batch_size, seq_len, hidden_size*2]
        context_vector = context_vector[:, -1, :]  # 取最后一个时间步的输出
        
        # 输出预测
        linear_out = self.linear(context_vector)
        output = self.sigmoid(linear_out)
        return output

# 数据预处理函数
def preprocess_data(train_path, test_path):
    # 加载数据
    train_df = pd.read_csv(train_path)
    test_df = pd.read_csv(test_path)
    
    # 提取特征和标签
    X_train = train_df.iloc[:, 1:-1].values  # 特征
    y_train = train_df.iloc[:, -1].values    # 标签
    X_test = test_df.iloc[:, 1:].values      # 测试特征
    
    # 特征标准化
    scaler = StandardScaler()
    X_train = scaler.fit_transform(X_train)
    X_test = scaler.transform(X_test)
    
    # 构建序列数据（这里使用滑动窗口）
    # 对于本数据集，我们将每个样本视为一个序列
    X_train = X_train.reshape(-1, 1, X_train.shape[1])
    X_test = X_test.reshape(-1, 1, X_test.shape[1])
    
    # 转换为PyTorch张量
    X_train = torch.tensor(X_train, dtype=torch.float32)
    y_train = torch.tensor(y_train, dtype=torch.float32).reshape(-1, 1)
    X_test = torch.tensor(X_test, dtype=torch.float32)
    
    return X_train, y_train, X_test, scaler

# 训练函数
def train_model(model, X_train, y_train, num_epochs=100, batch_size=32, learning_rate=0.001):
    # 创建数据加载器
    dataset = torch.utils.data.TensorDataset(X_train, y_train)
    dataloader = torch.utils.data.DataLoader(dataset, batch_size=batch_size, shuffle=True)
    
    # 定义优化器和损失函数
    optimizer = optim.Adam(model.parameters(), lr=learning_rate)
    criterion = nn.BCELoss()  # 二分类交叉熵损失
    
    # 训练模型
    for epoch in range(num_epochs):
        epoch_loss = 0.0
        for batch_data, batch_labels in dataloader:
            optimizer.zero_grad()
            outputs = model(batch_data)
            loss = criterion(outputs, batch_labels)
            loss.backward()
            optimizer.step()
            epoch_loss += loss.item() * batch_data.shape[0]
        epoch_loss /= len(dataset)
        print('Epoch [{}/{}], Loss: {:.4f}'.format(epoch+1, num_epochs, epoch_loss))
    
    return model

# 预测函数
def predict(model, X_test):
    model.eval()
    with torch.no_grad():
        outputs = model(X_test)
        predictions = (outputs >= 0.5).float()
    return predictions.numpy()

# 主函数
def main():
    # 数据路径
    train_path = 'd:\\fushi_work\\predict_Lottery_ticket_pytorch-master\\predict_Lottery_ticket_pytorch-master\\data_blood\\blood-train.csv'
    test_path = 'd:\\fushi_work\\predict_Lottery_ticket_pytorch-master\\predict_Lottery_ticket_pytorch-master\\data_blood\\blood-test.csv'
    
    # 预处理数据
    X_train, y_train, X_test, scaler = preprocess_data(train_path, test_path)
    
    # 初始化模型
    input_size = X_train.shape[2]
    model = BloodDonationLSTM(input_size=input_size)
    
    # 训练模型
    model = train_model(model, X_train, y_train)
    
    # 预测
    predictions = predict(model, X_test)
    
    # 保存预测结果
    test_df = pd.read_csv(test_path)
    test_df['Made Donation in March 2007'] = predictions
    test_df.to_csv('blood_prediction_results.csv', index=False)
    print("预测结果已保存到 blood_prediction_results.csv")

if __name__ == "__main__":
    main()
```

## 5. 评估指标

### 5.1 模型评估指标
| 指标 | 描述 | 计算方法 |
|------|------|----------|
| 准确率 | 正确预测的样本数占总样本数的比例 | (TP + TN) / (TP + TN + FP + FN) |
| 精确率 | 预测为正例的样本中实际为正例的比例 | TP / (TP + FP) |
| 召回率 | 实际为正例的样本中被正确预测的比例 | TP / (TP + FN) |
| F1分数 | 精确率和召回率的调和平均值 | 2 * (精确率 * 召回率) / (精确率 + 召回率) |
| AUC-ROC | 接收器操作特征曲线下的面积 | 通过计算不同阈值下的真阳性率和假阳性率 |

### 5.2 评估方法
1. 使用训练集和验证集进行模型训练和调优
2. 使用测试集评估模型性能
3. 计算各项评估指标，分析模型表现
4. 绘制ROC曲线，评估模型的分类能力
